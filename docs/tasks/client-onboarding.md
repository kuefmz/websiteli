# Client Onboarding

## Objective

Implement a premium reservation-based onboarding flow for Websiteli.

## User Journey

1.  Homepage CTA → `/reserve`
2.  Reservation form
3.  Thank-you page
4.  Manual invoice by email
5.  Customer pays via bank transfer
6.  Mark project as Paid
7.  Send onboarding form
8.  Customer completes questionnaire
9.  Customer books Calendly strategy call
10. Build, review and launch website.

## Pages

-   `/reserve`
-   `/reserve/thank-you`
-   `/onboarding`
-   `/admin/projects` (internal)

## Design

Reuse existing components, branding and layouts. Use black/white/red
palette. Responsive, accessible, SEO-friendly.

## Acceptance Criteria

-   No payment gateway.
-   Static-friendly.
-   Build passes.
