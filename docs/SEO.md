# SEO

This site is built as a static Astro site with localized routes, canonical URLs, hreflang alternates, structured data, sitemap generation, and performance-oriented assets.

## Metadata

Global metadata is set in `src/layouts/Layout.astro`:

- `<title>`
- meta description
- robots
- theme color and icon metadata
- canonical URL
- hreflang alternates
- OpenGraph
- Twitter card
- JSON-LD structured data

Page-level title/description comes from locale JSON or content registries.

## Canonical URLs

- Canonicals are clean URLs without UTM parameters.
- Root `/` canonical points to `/en/` redirect target.
- Localized pages canonicalize to `/:locale/.../`.
- Article canonical path is `/:locale/blog/:slug/`.

Do not include campaign query parameters in canonical URLs.

## Hreflang

`Layout.astro` renders alternate links from `getAlternates(page)` or article-specific alternates.

Current `x-default` points to `/en/`.

Risk: article language switcher in header goes to blog index because header receives `page="blog"`. Article metadata still has article-specific alternates.

## Schema.org

Global schema in `Layout.astro`:

- `Organization`
- `WebSite`
- `LocalBusiness`
- `Service`
- site-wide `FAQPage`

Article schema in `src/pages/[locale]/blog/[slug]/index.astro`:

- `BlogPosting`
- `BreadcrumbList`
- `FAQPage` when `post.faqs` exists

Portfolio cards include `CreativeWork` microdata.

## OpenGraph And Twitter Cards

Global OG/Twitter tags use localized title/description and `/assets/websiteli-og.png`.

`og:type` defaults to `website`. Article pages pass `ogType="article"`.

## Image Optimization

Current assets are static files in `public/assets/`:

- `websiteli-og.png`
- `swiss-ai-websites-hero.png`
- `shopify-consultant-portfolio-demo-preview.svg`

Images include explicit width/height in major components. Blog article hero uses eager loading; list/portfolio previews use lazy loading.

## Heading Hierarchy

- Each page should have a single H1.
- Section headings use H2.
- Cards commonly use H3.
- Blog body sections render H2s from each post file's body content.

Do not insert decorative headings that disrupt semantic structure.

## URL Conventions

- Locale prefix is mandatory for public localized pages.
- Current page slugs: `services-pricing`, `portfolio`, `blog`, `about`, `contact`.
- Legacy slugs redirect: `services`, `packages`, `demos`, `example-projects`.
- Service detail pages use `/:locale/services/:service/` and are generated from `src/content/services.ts`.
- Industry detail pages use `/:locale/industries/:industry/` and are generated from `src/content/industries.ts`.
- Blog article slug is currently shared across locales.

## Blog Optimization

Blog articles should include:

- Unique localized meta title and description.
- Clean slug.
- Localized H1 and headings.
- Reading time, publication date, updated date, and structured author metadata.
- Tags and category.
- Sticky TOC and progress bar.
- Inline and bottom conversion CTAs.
- Related internal links.
- FAQ schema where applicable.

## Sitemap

Configured in `astro.config.mjs` using `@astrojs/sitemap`.

The sitemap filter excludes legacy redirect-only pages and redirect namespaces while allowing generated `/services/:service/` SEO pages.

`public/robots.txt` points crawlers to:

- `https://websiteli.ch/sitemap-index.xml`
- `https://websiteli.ch/rss.xml`

## RSS

RSS is generated by `src/pages/rss.xml.ts`.

Known limitation: it currently includes English blog posts only.

## Technical SEO Checklist

Before merging SEO-sensitive changes:

- Run `npm run build`.
- Inspect generated HTML for canonical, hreflang, title, description, OG/Twitter tags.
- Confirm sitemap still excludes legacy redirects.
- Confirm article schema validates conceptually.
- Confirm no public roadmap/unpublished content appears.
- Confirm root and legacy redirects preserve query strings.
- Confirm Lighthouse SEO score does not regress when browser tooling is available.

## Performance Targets

Target Lighthouse scores:

- Performance: greater than 95.
- Accessibility: greater than 95.
- Best Practices: greater than 95.
- SEO: as close to 100 as possible.

There is no automated Lighthouse script currently. See `KNOWN_LIMITATIONS.md`.
